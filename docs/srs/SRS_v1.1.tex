% --- SECTION 1: INTRODUCTION ---
\section{Introduction}

\subsection{Purpose}
The purpose of this Software Requirements Specification (SRS) is to outline the functional and external interface requirements for the Lunar Rocks! system. It defines the scope, expected behavior, and constraints of the software for development.

\subsection{Intended Audience}
This document is intended for:
\begin{itemize}
    \item Software developers working on frontend, backend, or database components.
    \item Project managers overseeing progress and requirements.
    \item QA testers evaluating whether the system meets specifications.
\end{itemize}

\subsection{Overview of the Software}
Lunar Rocks! is a web-based citizen science tool enabling users to analyze lunar images from NASA missions. The application allows users to register, log in, scout images, size rocks, classify rock shapes, and submit results to a centralized scientific dataset.

\subsection{Definitions, Acronyms, Abbreviations}
\begin{itemize}
    \item \textbf{SRS} — Software Requirements Specification  
    \item \textbf{UI} — User Interface  
    \item \textbf{API} — Application Programming Interface  
    \item \textbf{DB} — Database  
    \item \textbf{NASA} — National Aeronautics and Space Administration  
\end{itemize}

\subsection{Software Interfaces}
The system interacts with:
\begin{itemize}
    \item A Java backend service running on Tomcat.
    \item A PostgreSQL database containing user, image, and rock classification data.
    \item NASA-hosted image datasets (future expansion).
\end{itemize}

\subsection{Communications Interfaces}
\begin{itemize}
    \item HTTPS for secure client-server communication.
    \item Email contact for users needing support.
\end{itemize}

\newpage

\subsection{Functional Requirements}
\begin{itemize}
    \item The system shall allow users to register for an account using an email and password.
    \item The system shall allow registered users to log in and log out.
    \item The system shall restrict access to task pages to authenticated users.
    \item The system shall provide an authenticated dashboard page that lists available tasks and allows users to begin a task.
\end{itemize}

\subsection{External Interface Requirements}

\subsubsection{Login and Registration UI}
The system shall provide a login/registration interface that:
\begin{itemize}
    \item collects credentials securely,
    \item provides clear success/failure feedback, and
    \item redirects authenticated users to the dashboard.
\end{itemize}

\subsubsection{Dashboard UI}
The system shall provide a dashboard interface that:
\begin{itemize}
    \item is accessible only after login,
    \item displays entry points to scouting, sizing, and classification, and
    \item provides basic user navigation (e.g., return to Home, log out).
\end{itemize}

\subsection{Dependencies and Libraries}
\begin{itemize}
    \item Password hashing library (e.g., bcrypt or equivalent)
    \item Authentication/session mechanism (session cookies or token-based auth)
\end{itemize}

\section{Legal and Ethical Considerations}

\subsection{Data Privacy}
\begin{itemize}
    \item User accounts require only minimal personal information.
    \item Passwords must be securely hashed.
    \item No user-generated scientific data is linked to identifiable personal information.
\end{itemize}

\subsection{Accessibility}
The system follows WCAG and USWDS guidance:
\begin{itemize}
    \item Adequate color contrast.
    \item Keyboard navigability.
    \item Descriptive labels and alt-text for all images.
\end{itemize}

\subsection{Ethical Use}
\begin{itemize}
    \item Citizen science contributions must be used responsibly.
    \item Data must be preserved with integrity for scientific use.
\end{itemize}
